# Refactor Clean

모든 단계에서 테스트를 검증하며 데드 코드를 안전하게 식별하고 제거합니다.

## 1단계: 데드 코드 감지

프로젝트 유형에 따라 분석 도구 실행:

| 도구 | 탐지 대상 | 명령어 |
|------|----------|---------|
| knip | 사용하지 않는 export, 파일, 의존성 | `npx knip` |
| depcheck | 사용하지 않는 npm 의존성 | `npx depcheck` |
| ts-prune | 사용하지 않는 TypeScript export | `npx ts-prune` |
| vulture | 사용하지 않는 Python 코드 | `vulture src/` |
| deadcode | 사용하지 않는 Go 코드 | `deadcode ./...` |
| cargo-udeps | 사용하지 않는 Rust 의존성 | `cargo +nightly udeps` |

사용 가능한 도구가 없으면 Grep으로 import가 없는 export를 찾습니다:
```
# export를 찾은 후 어디에서도 import되지 않는지 확인
```

## 2단계: 발견 사항 분류

안전도 계층으로 분류:

| 계층 | 예시 | 조치 |
|------|------|------|
| **안전** | 사용하지 않는 유틸리티, 테스트 헬퍼, 내부 함수 | 자신 있게 삭제 |
| **주의** | 컴포넌트, API 라우트, 미들웨어 | 동적 import나 외부 소비자 없는지 확인 |
| **위험** | 설정 파일, 진입점, 타입 정의 | 건드리기 전 철저히 조사 |

## 3단계: 안전 삭제 루프

각 안전(SAFE) 항목에 대해:

1. **전체 테스트 스위트 실행** — 기준선 확립 (모두 통과)
2. **데드 코드 삭제** — Edit 도구로 정밀 제거
3. **테스트 스위트 재실행** — 아무것도 깨지지 않았는지 확인
4. **테스트 실패 시** — 즉시 `git checkout -- <file>`로 되돌리고 이 항목 건너뜀
5. **테스트 통과 시** — 다음 항목으로 이동

## 4단계: 주의(CAUTION) 항목 처리

주의 항목 삭제 전:
- 동적 import 검색: `import()`, `require()`, `__import__`
- 문자열 참조 검색: 설정 파일의 라우트명, 컴포넌트명
- 공개 패키지 API에서 export되는지 확인
- 외부 소비자 없는지 확인 (배포된 경우 의존자 확인)

## 5단계: 중복 통합

데드 코드 제거 후 다음 항목 탐색:
- 거의 동일한 함수 (80% 이상 유사) — 하나로 병합
- 중복 타입 정의 — 통합
- 가치를 추가하지 않는 래퍼 함수 — 인라인으로 변경
- 목적 없는 재export — 간접 참조 제거

## 6단계: 요약

결과 보고:

```
데드 코드 정리
──────────────────────────────
삭제:   사용하지 않는 함수 12개
        사용하지 않는 파일 3개
        사용하지 않는 의존성 5개
건너뜀: 2개 항목 (테스트 실패)
절약:   약 450줄 제거
──────────────────────────────
모든 테스트 통과 ✅
```

## 규칙

- **테스트 실행 전에 절대 삭제하지 않음**
- **한 번에 하나씩 삭제** — 원자적 변경으로 롤백 용이
- **불확실하면 건너뜀** — 프로덕션을 망가뜨리는 것보다 데드 코드를 유지하는 것이 나음
- **정리 중 리팩토링하지 않음** — 관심사 분리 (먼저 정리, 이후 리팩토링)
