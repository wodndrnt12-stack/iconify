# 빌드 및 수정

빌드 오류와 타입 오류를 최소한의 안전한 변경으로 점진적으로 수정합니다.

## 1단계: 빌드 시스템 감지

프로젝트의 빌드 도구를 식별하고 빌드를 실행합니다:

| 식별자 | 빌드 명령 |
|-----------|---------------|
| `package.json` with `build` script | `npm run build` or `pnpm build` |
| `tsconfig.json` (TypeScript only) | `npx tsc --noEmit` |
| `Cargo.toml` | `cargo build 2>&1` |
| `pom.xml` | `mvn compile` |
| `build.gradle` | `./gradlew compileJava` |
| `go.mod` | `go build ./...` |
| `pyproject.toml` | `python -m py_compile` or `mypy .` |

## 2단계: 오류 파싱 및 그룹화

1. 빌드 명령을 실행하고 stderr를 캡처합니다
2. 파일 경로별로 오류를 그룹화합니다
3. 의존성 순서로 정렬합니다 (로직 오류 전에 import/타입 오류를 먼저 수정)
4. 진행 상황 추적을 위해 전체 오류 수를 계산합니다

## 3단계: 수정 루프 (오류 하나씩)

각 오류에 대해:

1. **파일 읽기** — Read 도구를 사용하여 오류 컨텍스트 확인 (오류 주변 10줄)
2. **진단** — 근본 원인 파악 (누락된 import, 잘못된 타입, 문법 오류)
3. **최소한으로 수정** — 오류를 해결하는 가장 작은 변경에 Edit 도구 사용
4. **빌드 재실행** — 오류가 사라지고 새 오류가 발생하지 않는지 확인
5. **다음으로 이동** — 남은 오류 계속 처리

## 4단계: 가드레일

다음 경우 중지하고 사용자에게 확인합니다:
- 수정으로 인해 **해결한 것보다 더 많은 오류가 발생**하는 경우
- **3번 시도 후에도 동일한 오류가 지속**되는 경우 (더 깊은 문제일 가능성)
- 수정이 **아키텍처 변경이 필요**한 경우 (단순 빌드 수정이 아닌)
- 빌드 오류가 **누락된 의존성**에서 발생하는 경우 (`npm install`, `cargo add` 등이 필요)

## 5단계: 요약

결과를 표시합니다:
- 수정된 오류 (파일 경로 포함)
- 남은 오류 (있는 경우)
- 새로 발생한 오류 (0이어야 함)
- 미해결 문제에 대한 다음 단계 제안

## 복구 전략

| 상황 | 조치 |
|-----------|--------|
| 누락된 모듈/import | 패키지 설치 여부 확인; 설치 명령 제안 |
| 타입 불일치 | 두 타입 정의를 읽고 더 좁은 타입으로 수정 |
| 순환 의존성 | import 그래프로 순환 파악; 추출 제안 |
| 버전 충돌 | `package.json` / `Cargo.toml`의 버전 제약 확인 |
| 빌드 도구 잘못된 설정 | 설정 파일 읽기; 작동하는 기본값과 비교 |

안전을 위해 오류를 하나씩 수정합니다. 리팩터링보다 최소 변경을 선호합니다.
