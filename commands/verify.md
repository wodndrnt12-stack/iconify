# 검증 명령어

현재 코드베이스 상태에 대한 포괄적인 검증을 실행합니다.

## 지침

정확히 다음 순서로 검증을 실행합니다:

1. **빌드 확인**
   - 이 프로젝트의 빌드 명령어 실행
   - 실패하면 오류를 보고하고 중지

2. **타입 확인**
   - TypeScript/타입 체커 실행
   - file:line과 함께 모든 오류 보고

3. **린트 확인**
   - 린터 실행
   - 경고와 오류 보고

4. **테스트 스위트**
   - 모든 테스트 실행
   - 통과/실패 수 보고
   - 커버리지 비율 보고

5. **Console.log 감사**
   - 소스 파일에서 console.log 검색
   - 위치 보고

6. **Git 상태**
   - 커밋되지 않은 변경사항 표시
   - 마지막 커밋 이후 수정된 파일 표시

## 출력

간결한 검증 보고서 생성:

```
검증: [통과/실패]

빌드:    [정상/실패]
타입:    [정상/X개 오류]
린트:    [정상/X개 이슈]
테스트:  [X/Y개 통과, Z% 커버리지]
시크릿:  [정상/X개 발견]
로그:    [정상/X개 console.log]

PR 준비: [예/아니오]
```

치명적 이슈가 있으면 수정 제안과 함께 목록화합니다.

## 인수

$ARGUMENTS 값:
- `quick` - 빌드 + 타입만 확인
- `full` - 모든 검사 (기본값)
- `pre-commit` - 커밋에 관련된 검사
- `pre-pr` - 보안 스캔을 포함한 전체 검사
