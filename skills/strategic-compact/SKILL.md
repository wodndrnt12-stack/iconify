# 전략적 압축 Skill

임의적인 자동 압축에 의존하는 대신 워크플로우의 전략적 지점에서 수동 `/compact`를 제안합니다.

## 활성화 시점

- 컨텍스트 한계(200K+ 토큰)에 가까워지는 긴 세션 실행 시
- 다단계 작업 진행 시 (리서치 → 계획 → 구현 → 테스트)
- 같은 세션 내에서 관련 없는 작업 간 전환 시
- 주요 마일스톤 완료 후 새 작업 시작 시
- 응답이 느려지거나 일관성이 떨어질 때 (컨텍스트 압박)

## 왜 전략적 압축인가?

자동 압축은 임의적 지점에서 트리거됩니다:
- 종종 작업 중간에, 중요한 컨텍스트를 잃음
- 논리적 작업 경계에 대한 인식 없음
- 복잡한 다단계 작업을 중단할 수 있음

논리적 경계에서의 전략적 압축:
- **탐색 후, 실행 전** — 리서치 컨텍스트 압축, 구현 계획 유지
- **마일스톤 완료 후** — 다음 단계를 위한 새로운 시작
- **주요 컨텍스트 전환 전** — 다른 작업 전 탐색 컨텍스트 정리

## 작동 방식

`suggest-compact.js` 스크립트가 PreToolUse (Edit/Write)에서 실행되어:

1. **도구 호출 추적** — 세션에서 도구 호출 횟수 계산
2. **임계값 감지** — 구성 가능한 임계값에서 제안 (기본값: 50회 호출)
3. **주기적 알림** — 임계값 이후 매 25회 호출마다 알림

## 훅 설정

`~/.claude/settings.json`에 추가:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      },
      {
        "matcher": "Write",
        "hooks": [{ "type": "command", "command": "node ~/.claude/skills/strategic-compact/suggest-compact.js" }]
      }
    ]
  }
}
```

## 구성

환경 변수:
- `COMPACT_THRESHOLD` — 첫 번째 제안 전 도구 호출 수 (기본값: 50)

## 압축 결정 가이드

언제 압축할지 결정하기 위한 표:

| 단계 전환 | 압축? | 이유 |
|-----------------|----------|-----|
| 리서치 → 계획 | 예 | 리서치 컨텍스트는 방대함; 계획이 정제된 출력 |
| 계획 → 구현 | 예 | 계획이 TodoWrite 또는 파일에 있음; 코드를 위한 컨텍스트 확보 |
| 구현 → 테스트 | 경우에 따라 | 테스트가 최근 코드를 참조하면 유지; 초점 전환 시 압축 |
| 디버깅 → 다음 기능 | 예 | 디버그 추적이 관련 없는 작업의 컨텍스트를 오염시킴 |
| 구현 중간 | 아니요 | 변수 이름, 파일 경로, 부분 상태를 잃는 것은 비용이 큼 |
| 실패한 접근 후 | 예 | 새 접근 시도 전 막힌 추론 정리 |

## 압축 후 남는 것

무엇이 지속되는지 이해하면 자신 있게 압축할 수 있습니다:

| 지속됨 | 사라짐 |
|----------|------|
| CLAUDE.md 지침 | 중간 추론 및 분석 |
| TodoWrite 작업 목록 | 이전에 읽은 파일 내용 |
| 메모리 파일 (`~/.claude/memory/`) | 다단계 대화 컨텍스트 |
| Git 상태 (커밋, 브랜치) | 도구 호출 기록 및 횟수 |
| 디스크의 파일 | 구두로 언급된 세밀한 사용자 선호사항 |

## 모범 사례

1. **계획 후 압축** — 계획이 TodoWrite에 확정되면 새로운 시작을 위해 압축
2. **디버깅 후 압축** — 계속하기 전 오류 해결 컨텍스트 정리
3. **구현 중간에 압축하지 않기** — 관련 변경을 위한 컨텍스트 보존
4. **제안 내용 읽기** — 훅이 *언제*를 알려주고, *할지 말지*는 사용자가 결정
5. **압축 전 저장** — 압축 전 중요한 컨텍스트를 파일이나 메모리에 저장
6. **요약과 함께 `/compact` 사용** — 사용자 정의 메시지 추가: `/compact 다음으로 인증 미들웨어 구현에 집중`
