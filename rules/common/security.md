# 보안 지침

## 필수 보안 체크

모든 커밋 전:
- [ ] 하드코딩된 시크릿 없음 (API 키, 비밀번호, 토큰)
- [ ] 모든 사용자 입력 유효성 검사됨
- [ ] SQL 인젝션 방지 (파라미터화된 쿼리)
- [ ] XSS 방지 (HTML 살균 처리)
- [ ] CSRF 보호 활성화됨
- [ ] 인증/인가 확인됨
- [ ] 모든 엔드포인트에 속도 제한 적용됨
- [ ] 오류 메시지가 민감한 데이터를 노출하지 않음

## 시크릿 관리

- 소스 코드에 시크릿 절대 하드코딩 금지
- 항상 환경 변수 또는 시크릿 매니저 사용
- 시작 시 필수 시크릿 존재 여부 확인
- 노출되었을 수 있는 시크릿 즉시 교체

## 보안 대응 프로토콜

보안 이슈 발견 시:
1. 즉시 중지
2. **security-reviewer** 에이전트 사용
3. 계속 진행하기 전에 CRITICAL 이슈 수정
4. 노출된 시크릿 교체
5. 유사 이슈에 대해 전체 코드베이스 검토
