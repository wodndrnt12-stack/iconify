# 개발 워크플로우

> 이 파일은 [common/git-workflow.md](./git-workflow.md)를 확장하여 git 작업 이전에 이루어지는 전체 기능 개발 프로세스를 설명한다.

기능 구현 워크플로우는 리서치, 계획 수립, TDD, 코드 리뷰, 그리고 git 커밋으로 이어지는 개발 파이프라인을 기술한다.

## 기능 구현 워크플로우

0. **리서치 & 재사용** _(새 구현 전 필수)_
   - **GitHub 코드 검색 우선:** 새로운 코드를 작성하기 전에 `gh search repos`와 `gh search code`를 실행하여 기존 구현, 템플릿, 패턴을 찾는다.
   - **리서치용 Exa MCP:** 계획 단계에서 광범위한 리서치, 데이터 수집, 선행 사례 발견을 위해 `exa-web-search` MCP를 사용한다.
   - **패키지 레지스트리 확인:** 유틸리티 코드를 직접 작성하기 전에 npm, PyPI, crates.io 등 레지스트리를 검색한다. 직접 구현보다 검증된 라이브러리를 선호한다.
   - **적용 가능한 구현 검색:** 문제의 80% 이상을 해결하고 포크, 포팅, 래핑할 수 있는 오픈소스 프로젝트를 찾는다.
   - 요구사항을 충족하는 검증된 접근법의 도입 또는 포팅을 새 코드 작성보다 우선한다.

1. **먼저 계획 수립**
   - **planner** 에이전트를 사용하여 구현 계획 작성
   - 코딩 전 계획 문서 생성: PRD, 아키텍처, 시스템 설계, 기술 문서, 태스크 목록
   - 의존성과 위험 요소 식별
   - 단계별로 분해

2. **TDD 접근법**
   - **tdd-guide** 에이전트 사용
   - 먼저 테스트 작성 (RED)
   - 테스트를 통과하도록 구현 (GREEN)
   - 리팩터링 (IMPROVE)
   - 80% 이상 커버리지 확인

3. **코드 리뷰**
   - 코드 작성 직후 **code-reviewer** 에이전트 사용
   - CRITICAL 및 HIGH 이슈 처리
   - 가능한 경우 MEDIUM 이슈 수정

4. **커밋 & 푸시**
   - 상세한 커밋 메시지
   - 컨벤셔널 커밋 형식 준수
   - 커밋 메시지 형식과 PR 프로세스는 [git-workflow.md](./git-workflow.md) 참조
